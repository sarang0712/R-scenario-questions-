🏆 BIG CONCEPT YOU JUST LEARNED
⭐ R uses Lexical Scoping

Variables belong to the environment where they are created.

Child functions can SEE parent variables
—but cannot change them (normally).

🧃 Ultra-Simple Analogy

Think of Russian dolls 🪆

Big doll (global)
  inside → outer
    inside → middle
      inside → inner
        inside → middle_inner

Each doll has its own X sticker 🏷️

⭐ Key Rule (Very Important)

R searches variables in this order:
Child → Parent → Grandparent → Global → Base

⚠️ Reverse is NOT true

Parent CANNOT see child variables.

🎯 Interview-Ready Definition

👉 “R uses lexical scoping — a function looks for variables in the environment where it was defined, not where it is called.”


Let’s walk through your code step by step:

f1 <- function () {
  
  x <<- 5
  
  f2 <- function() {
    x <- 10
    x + 1
  }
  f2()
}
f1()

🔎 Why is x treated as 10 inside f2()?

Because of R’s lexical scoping rules.

Inside f2() you wrote:

x <- 10

This creates a new local variable x inside f2().

When you then compute:

x + 1

R looks for x:

First inside f2() → finds x <- 10

So it uses 10

Returns 11

It never looks outside to the x <<- 5 because a local variable named x already exists.

🔁 What does <<- actually do?

This line:


x <<- 5

Assigns x = 5 in the parent/global environment (not inside f1).

But inside f2(), you created a new local x, which hides the outer one.

So this is happening:

Global: x = 5   (created by <<-)

Inside f2:
    x = 10      (local variable)

The local variable masks the outer one.

🧠 Key Concept

<- → creates/updates variable in current environment

<<- → updates variable in parent/global environment

But local variables always override outer variables if they have the same name.
